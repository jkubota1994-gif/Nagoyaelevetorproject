## 【今後の作業用】手動で追加するエレベーターデータに必要な情報

現状のOSMデータに不足しているエレベーター情報を手動でマップに追加していく場合、以下の情報が必要です。

### 1. 調査・収集していただきたいデータ項目（ExcelやCSVでまとめる場合）

調査の際は、Excelやスプレッドシート等で以下の情報をリストアップしていただくのが最も簡単です。（後でプログラムを使って一括でマップ用データに変換できます）

*   **駅名 (Station):** どの駅に属するエレベーターか（例: 栄）
*   **緯度 (Latitude):** （例: 35.1698） ※Googleマップで対象の場所を右クリックすると一番上に表示・コピーできます
*   **経度 (Longitude):** （例: 136.9064）
*   **場所や説明 (Location / Description):** どこからどこへ繋がっているかなどの補足（例: 「地上〜コンコース間」「東山線 中改札外付近」など）
*   **管理用ID:** （例: "manual_sakae_01" などユニークな英数字） ※省略しても後から自動で連番を振ることは可能です。

### 2. マップが直接読み込むデータのフォーマット (参考: GeoJSON形式)

実際にマップ上にピンを立てるために `data/all-elevators.geojson` のファイル内に追記されるデータフレームは以下のようになります。

```json
{
  "type": "Feature",
  "geometry": {
    "type": "Point",
    // ※注意点: GeoJSONの仕様により、座標は必ず [経度(Lon), 緯度(Lat)] の順番になります
    "coordinates": [136.9064, 35.1698] 
  },
  "properties": {
    "id": "manual_sakae_01",
    "station": "栄",
    "location": "オアシス21連絡口",
    "description": "地上〜改札階",
    "status": "稼働中"
  }
}
```

Googleマップ等で座標を調べる際は、「緯度, 経度」の順でクリップボードにコピーされることが多いです。マップのデータ（GeoJSON）として登録する際には順番を `[経度, 緯度]` （X, Yの順番）に逆にする必要がある点にのみご注意ください。
